# Data Bridge v0.2.2 — live Korean verification, 3-party review hardening, CI/CD

> Status: **deployed to production** 2026-07-23 (revision `databridge-00003-kvr`,
> build `aa2e252d`). Scope: closes the v0.2.1 follow-ups (§4 live gate, §5 production
> note) and moves the project from manual deploys to continuous deployment.

## 1. What shipped

Three change sets, individually reviewed and merged:

- **[PR #4](https://github.com/ParkHwan/Data-Bridge/pull/4) — Korean golden items.**
  New corpus doc `samples/docs/security-review-ko.md` written with josa-suffixed
  keyword forms (`침투 테스트는`, `패치를`) plus two Korean golden questions
  (DG-006/DG-007) using bare-form keywords — the english FTS parser cannot bridge that
  morphology gap, so these items exercise the v0.2.1 trigram RRF source live.
  Golden set is now 7 items.
- **[PR #5](https://github.com/ParkHwan/Data-Bridge/pull/5) — Cloud Build CI/CD**,
  hardened per the 3-party review (§3):
  `main` push → gates (ruff / mypy / pytest incl. the 8 store integration tests
  against a throwaway `pgvector/pgvector:pg16` on the build network) → locked-deps
  image build (`uv sync --locked`, gate and image share one `uv.lock` graph) → push →
  **schema-before-code** (`databridge-migrate` Cloud Run job runs
  `scripts/migrate.py` → `ensure_schema()`) → deploy by resolved `@sha256` digest.
  Build identity is a dedicated least-privilege SA (`databridge-build@`); all infra
  (SA, IAM, jobs, GitHub connection, trigger) is reproducible via
  `scripts/setup_cicd.sh`.
- **[PR #6](https://github.com/ParkHwan/Data-Bridge/pull/6) — deterministic candidate
  ordering.** The three hybrid candidate queries gained stable secondary sort keys
  (`space_key, chunk_id`): equal-score row order is planner-dependent in Postgres and
  fed directly into RRF ranks, so ties at the `candidate_k` boundary could previously
  flip which chunks (and citations) surfaced between identical runs.

## 2. Live verification (owner-run, real Gemini 2.5 Flash + Vertex embeddings)

Closing the v0.2.1 §4 pending item — the golden gate now includes Korean:

```
[PASS] DG-001 … DG-005  (English, unchanged — no regression vs v0.2.0/v0.2.1 baseline)
[PASS] DG-006 kw=1.00 src=O   A: 외부 침투 테스트는 분기마다 1회 …
[PASS] DG-007 kw=1.00 src=O   A: 심각 등급 취약점은 발견 후 48시간 이내에 패치를 배포 …
summary: keyword_hit=1.000  source_hit=7/7
```

Store-level proof that the trigram source (not just multilingual vectors) does the
recall: query `"패치 기한"` returns the Korean chunk with `fts_rank=None,
trgm_rank=1` — pure trigram recovery of a josa-suffixed keyword.

## 3. 3-party review

PRs #3/#4/#5 were independently reviewed by three agents (Claude, Antigravity,
Codex); 11 findings, all verified against the code and dispositioned — 6 fixed
before/at merge, 2 owner decisions open, 3 backlog. Full log:
[docs/design/review-log-v0.2.1-ci.md](../design/review-log-v0.2.1-ci.md).
Highlights fixed: gate `set -e`, lock-graph mismatch between gate and image,
integration tests skipping in CI, mutable-tag deploys, default-SA build identity,
schema/code deploy ordering, RRF tie non-determinism.
